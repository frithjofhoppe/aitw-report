\section{Problemstellung}

Eine Nutzung der verschiedenen Cloud-Plattformen bedingt, dass eine grundlegende Struktur existiert auf welche Anwender innerhalb der \ac{bv} aufbauen können. Das \ac{bit} strebt hierbei die Anwendung der Konzepte hinter dem Begriff \ac{lz} an. Die Annahme ist dass eine \ac{lz} eine durch den Plattformprovider vordefinierten Struktur bezeichnet, in welcher die effektiven Angebote bezogen werden können.

Aus der bisherigen Erfahrung \hl{hier verweis auf vorarbeit} unterschieden sich die Plattformprovider in der Art und Weise wie ihre Plattform strukturiert ist. Idee oder Konzepte werden zwar häufig zwischen den Providern geteilt, die Ausgestaltung unterscheidet sich jedoch. Für einen grösseren Anwender wie das \ac{bit} bedeutet dies das es einen Ansatz braucht, mit dem eine Anwendung (bspw dem Konzept der \ac{lz}) über Grenzen von Provider hinweg möglich ist. Es ist somit trotzt der Unterschiedlichkeit eine Homogenisierung wünschenswert. Eine Vereinheitlichung ermöglicht es dem \ac{bit} einen Leitrahmen zu definieren welcher für die verschiedenen Projekte gilt, welche ihre Vorhaben auf der Plattform laufen lassen.

Diese Arbeit beschäftigt sich im Rahmen der aufgezeigten Problematik mit folgenden Fragestellungen

\textit{Welche Komponenten machen eine Landingzone für eine Hybrid-Multi-Cloud-Umgebung aus?}

\textit{Wie sieht eine geeignete technische Architektur einer Landingzone für eine Hybrid-Multi-Cloud-Umgebung aus und was muss sie erfüllen?}
